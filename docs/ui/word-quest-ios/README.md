# Word Quest iOS 风格 UI 设计稿

本目录保存 Word Quest 第一版核心学习闭环的 iOS 风格 UI 设计稿。设计方向参考 iOS / iPadOS 常见信息架构：大标题、分组列表、主操作按钮、底部标签栏、iPad 侧边栏和清晰的状态反馈。

## 页面清单

- `home-iphone.png`：iPhone 首页，覆盖今日任务、学习路线、家长提醒和底部标签栏。
- `home-ipad.png`：iPad 首页，采用侧边栏加内容区结构。
- `study-quiz.png`：做题页，覆盖题目进度、听音入口、选项、答题反馈和下一题操作。
- `study-complete.png`：任务完成页，覆盖奖励反馈、学习统计、薄弱点和后续动作。
- `word-book.png`：错词与词表页，覆盖搜索、错词筛选、高频错词和错词复习入口。
- `family-dashboard.png`：家庭看板页，覆盖两个孩子的学习状态、词表导入、备份恢复和本地数据说明。
- `word-quest-ios-design.pdf`：以上页面的合并 PDF，便于整体浏览。

## 设计原则

- 以孩子当天学习任务为主线，首页只突出一个主动作：继续学习。
- 用分组列表承载路线、错词和家长管理，减少卡片堆叠带来的噪音。
- 用系统色语义表达状态：蓝色表示主操作，绿色表示完成或正确，橙色表示提醒，红色表示高优先级错词。
- iPhone 优先单列结构；iPad 使用侧边栏与详情区，保持与 iPadOS 的导航习惯一致。
- 本地优先的数据策略在家庭页明确呈现，避免用户误以为依赖云同步。

## 后续落地建议

- Flutter 实现时优先拆分为 `HomeScreen`、`StudyQuizScreen`、`StudyCompleteScreen`、`WordBookScreen`、`FamilyDashboardScreen`。
- 首页底部标签栏建议只承载一级导航：今天、闯关、词表、家庭。
- 做题页需要根据题型切换题干区和输入控件，但顶部进度、关闭、听音和底部主按钮应保持一致。
- 错词与词表页的筛选状态应与后续复习任务生成逻辑连接，避免只是静态列表。
