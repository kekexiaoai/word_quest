# 词途 iOS 风格 UI 设计稿

本目录保存「词途」第一版核心学习闭环的 iOS 风格 UI 设计稿。对外 App 名称为「词途」，副标题为「每天一小步，单词走得稳」，英文/内部代号保留为 Word Quest。

新版方向重点是降低首页负担和视觉噪音：首页只承载今日学习状态和下一步动作；孩子/家长切换移动到设置页；词表、错词和导入能力归到词表页；底部导航为今天、闯关、词表、设置。

## 页面清单

- `home-iphone.png`：iPhone 首页，覆盖今日任务、轻量统计、下一组练习和底部标签栏。
- `home-ipad.png`：iPad 首页，采用侧边栏加内容区结构，底部导航对应项改为设置。
- `study-quiz.png`：做题页，覆盖题目进度、听音入口、选项、答题反馈和下一题操作。
- `study-complete.png`：任务完成页，覆盖奖励反馈、学习统计、薄弱点和后续动作。
- `word-book.png`：词表页，覆盖当前词表、错词复习、搜索、最近练过和导入入口。
- `settings.png`：设置页，覆盖孩子/家长切换、家长管理、备份恢复和内部代号说明。
- `word-quest-ios-design.pdf`：新版页面的合并 PDF，便于整体浏览。

`family-dashboard.png` 是早期家庭页设计留档，不再作为新版底部导航页面使用。

## 设计原则

- 以孩子当天学习任务为主线，首页只突出一个主动作：继续学习。
- 控制字号、字重和卡片数量，避免界面呈现成放大版信息看板。
- 用分组列表承载词表、错词、设置和数据管理，减少首页卡片堆叠。
- 用系统色语义表达状态：绿色表示主学习动作和正确，橙色表示需要留意，靛蓝用于听音或辅助信息。
- iPhone 优先单列结构；iPad 使用侧边栏与详情区，保持与 iPadOS 的导航习惯一致。
- 本地优先的数据策略在设置页明确呈现，避免用户误以为依赖云同步。

## 后续落地建议

- Flutter 实现时优先拆分为 `HomeScreen`、`StudyQuizScreen`、`StudyCompleteScreen`、`WordBookScreen`、`SettingsScreen`。
- 首页底部标签栏只承载一级导航：今天、闯关、词表、设置。
- 设置页负责角色切换、孩子档案切换、家长管理入口和备份恢复入口。
- 做题页需要根据题型切换题干区和输入控件，但顶部进度、关闭、听音和底部主按钮应保持一致。
- 错词与词表页的筛选状态应与后续复习任务生成逻辑连接，避免只是静态列表。
