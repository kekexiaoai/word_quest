# Word Quest 关卡与宠物养成设计文档

## 背景

Word Quest 的第一版产品定位是“每日任务 + 闯关反馈”的本地背单词应用。仅有题目和正确率不足以和游戏体验竞争，因此需要把学习过程包装成更有目标感、进度感和情绪反馈的冒险体验。

本设计采用“冒险闯关型”为主线，后续融合轻量宠物养成。核心目标不是把 Word Quest 做成复杂游戏，而是用 Flutter 普通 2D UI、状态模型和轻动画，把背单词过程组织成孩子愿意每天回来的冒险循环。

## 设计目标

- 让孩子看到明确的下一步：今天要闯哪几关、当前卡在哪里、完成后会推进到哪里。
- 让学习反馈足够及时：答对、连击、完成小关、击败错词 Boss 都有可见反馈。
- 让错词不再等同于挫败：错词被包装成可以挑战和收服的目标，而不是孩子能力差的证明。
- 让宠物提供长期牵挂：孩子通过学习获得食物和能量，宠物陪伴闯关并逐步成长。
- 保持实现克制：第一版不做 3D、不做复杂小游戏、不做开放地图，不引入游戏引擎。

## 核心体验循环

```text
选择孩子档案
  ↓
进入今日冒险地图
  ↓
完成新词关、复习关、错词 Boss 关
  ↓
获得星星、食物、能量和宝箱
  ↓
喂养宠物，提升成长值
  ↓
地图推进，宠物陪伴状态变化
  ↓
第二天继续新的冒险节点
```

这个循环中，学习仍是主线，关卡和宠物是反馈层。系统不应该鼓励孩子为了喂宠物而乱点，也不应该让宠物状态制造负罪感。

## 关卡系统设计

### 关卡定位

关卡不是独立游戏关，而是对 `StudyTask` 的学习任务分组。现有任务中已经有新词、复习词、错词强化和多种题型，关卡系统负责把这些学习项转换成更有节奏的体验。

### V1 每日关卡结构

第一版建议每天生成 3 个学习关卡和 1 个结算关卡：

| 关卡 | 学习内容 | 情绪目标 | 主要奖励 |
| --- | --- | --- | --- |
| 新词热身关 | 新词选择题为主 | 先赢一次，降低进入成本 | 普通食物、星星 |
| 复习探索关 | 复习词选择题、拼写题、少量听音题 | 感到自己在巩固旧能力 | 能量、星星 |
| 错词 Boss 关 | 高频错词、近期错词、薄弱题型 | 把挫败改造成挑战 | 高级食物、宝箱进度 |
| 宝箱结算关 | 展示今日收获 | 完整收尾，强化成就感 | 宠物成长值、装饰碎片 |

### 关卡状态

每个关卡需要维护以下状态：

- `locked`：未解锁，孩子不能进入。
- `current`：当前可挑战。
- `completed`：已完成，可查看成绩和奖励。
- `reviewable`：可选状态，允许孩子重复挑战但不重复发放主要奖励。

每天默认按顺序解锁：完成新词热身关后解锁复习探索关，完成复习探索关后解锁错词 Boss 关，完成 Boss 关后进入宝箱结算。

### 关卡生成规则

关卡生成基于每日任务，不改变单词复习策略：

- 新词热身关使用 `StudyTaskType.newWords`，题型以 `englishToChinese` 和 `chineseToEnglish` 为主。
- 复习探索关使用 `StudyTaskType.review`，题型包含选择、拼写和少量听音。
- 错词 Boss 关使用 `StudyTaskType.mistakes`，优先安排连续答错、高频错、拼写薄弱和听辨薄弱的词。
- 如果当天没有错词，Boss 关可以转为“守关挑战”，从复习词中抽取掌握度较低的词。
- 如果当天任务量很少，仍保留关卡结构，但降低每关题量，避免空关卡。

### Boss 关设计

Boss 关是错词系统的情绪包装，不需要复杂战斗系统。

- 每个错词可以视为 Boss 的一段生命值。
- 答对一题削减生命值。
- 连续答对可以触发额外削减或宠物协助动画。
- 答错不回血，不惩罚孩子，只给出提示并安排稍后再试。
- 击败 Boss 后展示“收服了 X 个迷雾单词”之类的正向反馈。

这里的关键是把“我又错了”改成“这个词还没被我打败”。

### 即时反馈

每题完成后需要给短反馈：

- 答对：星星飞入进度条、宠物跳跃、连击数字增加。
- 答错：答案解释、宠物鼓励、该词进入本关稍后重试。
- 连击：展示小型连击奖励，但不制造排名压力。
- 关卡完成：节点点亮、路线推进、奖励落入背包。

反馈应该短、轻、可跳过，避免影响学习效率。

## 冒险地图设计

### 地图形式

V1 地图采用 2D 节点路线即可。Flutter 可以通过 `Stack`、`CustomPaint`、图片背景、节点按钮和简单动画实现，不需要游戏引擎。

地图页面至少包含：

- 当前冒险主题。
- 今日 3 个学习关卡节点。
- 宝箱结算节点。
- 孩子的宠物伙伴。
- 今日星星进度和宠物饱腹提示。

### 地图主题

第一版可以先使用一个固定主题，例如“森林冒险”。后续再扩展：

- 森林冒险：适合新手期，氛围轻松。
- 海岛航线：适合中期，强调探索。
- 星球远征：适合长期扩展，便于加入更多地图章节。

主题只影响视觉和文案，不影响学习算法。

## 宠物养成系统设计

### 宠物定位

宠物是孩子的冒险伙伴，不是需要持续照顾的压力来源。它通过孩子的学习成果获得食物和能量，陪孩子走地图、挑战 Boss、打开宝箱。

设计原则：

- 宠物不会死亡，不会生病，不会因为孩子没学习而惩罚孩子。
- 未学习时可以表达想念，但不能制造负罪感。
- 宠物成长主要来自真实学习行为，而不是单独点击。
- 宠物奖励要轻量，避免抢走背单词主线。

### V1 宠物属性

第一版只需要少量属性：

- `petId`：宠物标识。
- `name`：宠物名称，可默认生成，后续支持修改。
- `level`：成长等级。
- `growthPoints`：成长值。
- `satiety`：饱腹值。
- `mood`：心情状态。
- `equippedDecorationIds`：已装备装饰。
- `unlockedDecorationIds`：已解锁装饰。
- `lastFedAt`：上次喂食时间。

### 食物来源

食物必须和学习行为绑定：

| 学习行为 | 奖励 |
| --- | --- |
| 完成新词热身关 | 普通食物 |
| 完成复习探索关 | 能量饮品 |
| 完成错词 Boss 关 | 高级食物 |
| 连续答对达到阈值 | 额外小零食 |
| 连续学习多天 | 稀有装饰碎片 |
| 收服高频错词 | 成长碎片 |

奖励不要求复杂随机。V1 可以使用确定性奖励，避免孩子为了刷随机奖励重复无效操作。

### 喂食规则

- 孩子完成关卡后获得食物，食物进入今日奖励背包。
- 在首页或结算页可以一键喂食宠物。
- 喂食增加饱腹值和成长值。
- 饱腹值达到上限后仍可获得成长值，但不重复堆叠过多状态。
- 每天首次喂食可以触发宠物感谢文案。
- 重复挑战已完成关卡不重复发放主要食物，避免刷奖励。

### 成长规则

宠物通过成长值升级：

| 等级 | 解锁内容 |
| --- | --- |
| 1 | 默认宠物形象 |
| 2 | 探险围巾 |
| 3 | 小背包 |
| 4 | 地图脚印特效 |
| 5 | Boss 关协助动画 |

后续可以扩展更多装饰和形态，但 V1 不需要多宠物、多进化路线或复杂养成树。

### 宠物心情

心情用于表达陪伴感，不作为惩罚：

- `happy`：今天已完成学习或已喂食。
- `waiting`：今天还没有开始冒险。
- `cheering`：正在关卡中鼓励孩子。
- `celebrating`：完成 Boss 或宝箱结算。

避免使用“饥饿濒危”“伤心责备”等负向状态。

## 奖励系统设计

### 奖励层级

奖励分为三层：

1. 即时奖励：每题答对、连击、关卡进度。
2. 日内奖励：完成关卡获得星星、食物、能量。
3. 长期奖励：地图章节推进、宠物升级、装饰解锁。

### 奖励边界

- 奖励服务于学习，不以抽卡和随机稀有度作为核心吸引力。
- 不做付费道具，不做排行榜压力。
- 不鼓励无限刷题，重复挑战可以练习，但主要奖励每日限领。
- 家长端展示学习效果，不展示让孩子焦虑的惩罚指标。

## 页面流程建议

### 首页

- 显示当前孩子、今日冒险地图、宠物状态、今日任务入口。
- 地图节点显示锁定、当前、已完成状态。
- 宠物区域显示饱腹值、心情和可喂食提示。

### 关卡开始页

- 展示关卡名称、预计题量、主要奖励。
- 宠物给一句轻量鼓励文案。
- 进入答题挑战。

### 答题页

- 保持学习效率优先。
- 顶部展示关卡进度和星星。
- 宠物只作为轻反馈，不占用主要题目空间。

### 关卡完成页

- 展示正确率、获得星星、食物和能量。
- 点亮地图节点。
- 引导进入下一关或喂食宠物。

### 宝箱结算页

- 汇总今日成果。
- 展示宠物成长变化。
- 展示新解锁装饰或明天将进入的地图节点。

## 分阶段落地

### V1：最小完整闭环

- 每日任务拆成新词关、复习关、错词 Boss 关。
- 首页展示 2D 地图节点。
- 完成关卡获得星星和食物。
- 新增一个默认宠物。
- 支持喂食、成长值和等级。
- 宝箱页展示今日收获。

### V1.1：反馈增强

- 增加连击反馈。
- 增加宠物在答题页的轻动画。
- 增加装饰解锁和装备。
- 增加不同地图主题文案。

### V1.2：策略增强

- 根据薄弱类型生成专项 Boss。
- 根据年龄段调整关卡节奏。
- 支持家长配置每日任务强度。
- 支持复盘“本周收服的错词”。

## 不做事项

第一阶段明确不做：

- 3D 地图。
- 实时战斗。
- 复杂宠物繁殖或多形态进化。
- 抽卡系统。
- 排行榜。
- 社交系统。
- 付费道具。

这些能力会增加实现复杂度，并可能偏离“让孩子稳定背单词”的产品目标。

## 成功标准

- 孩子能理解今天要闯哪几关，并愿意点进当前关卡。
- 完成每日学习后，孩子能看到地图推进和宠物成长。
- 错词 Boss 关能降低孩子面对错词的挫败感。
- 家长仍能清楚看到学习结果，而不是只看到游戏奖励。
- 关卡和宠物系统不破坏现有每日任务、复习和错词逻辑。
