# Word Quest 产品设计文档

## 背景

Word Quest 是给两个孩子使用的背单词软件。目标用户覆盖小学高年级和初高中阶段，因此产品需要同时兼顾趣味性、效率和可持续复习。

项目采用本地优先策略：第一版不做账号和云同步，支持多孩子档案、本地学习数据、导入导出备份，并在架构上预留后续云同步能力。

## 产品定位

第一版定位为“每日任务 + 闯关反馈”的本地背单词应用。

- 每日任务是主线：系统根据词表、复习计划和错词情况生成当天任务。
- 闯关进度是反馈：孩子完成任务后获得星星或能量，推动地图节点。
- 家长端轻量管理：家长负责选择词表、导入词库、查看学习结果和备份数据。

## 目标用户

### 小学高年级孩子

- 需要更强的趣味性和即时反馈。
- 适合选择题、拼写题、少量听音题。
- 闯关、星星、连续学习等机制有助于保持动力。

### 初中和高中孩子

- 更关注效率、考试词表、复习节奏和错词清理。
- 需要清楚知道今日任务、正确率、薄弱词和复习安排。
- 后续版本可加入例句填空、词义辨析、词性变化和短语搭配。

### 家长

- 需要轻量了解两个孩子的学习状态。
- 需要能够管理内置词表和自定义词表。
- 不需要复杂控制台，重点是“今天学没学、哪里总错、数据能不能备份”。

## 第一版范围

### 必做能力

- 多孩子档案：两个孩子独立进度、错词、任务和关卡。
- 词库系统：内置分级词表和家长自定义导入。
- 每日任务：新词、复习词、错词强化。
- 基础题型：看英文选中文、看中文选英文、拼写单词、错词复习。
- 少量听音题：听音选词、听音拼写。
- 闯关反馈：完成任务后获得星星或能量，推进地图节点。
- 家长看板：今日完成、正确率、连续学习天数、高频错词。
- 本地数据：离线可用，支持备份导出和备份导入。

### 暂不实现

- 账号系统和云同步。
- 跟读评分。
- AI 对话练习。
- 小程序适配。
- 复杂考试题型。
- 家长远程提醒和推送通知。

## 核心学习流程

1. 孩子选择自己的档案。
2. 系统展示今日任务，包括新词、复习词和错词强化。
3. 孩子依次完成题目。
4. 系统根据答题结果更新单词熟练度、错词状态和下次复习时间。
5. 完成任务后，孩子获得星星或能量，推进闯关地图。
6. 家长可查看两个孩子的完成情况、正确率和薄弱词。

## 题型设计

### V1 题型

- 看英文选中文：用于快速建立单词和释义关联。
- 看中文选英文：用于反向回忆。
- 拼写单词：用于强化输出能力。
- 错词复习：优先处理近期答错和多次答错的词。
- 听音选词：使用语音或预置音频，让孩子根据发音选择单词。
- 听音拼写：听到单词后输入拼写，作为听写训练的基础版本。

### 题目生成原则

- 每个单词可以对应多种题型。
- 新词以选择题开始，降低首次学习难度。
- 熟悉后逐步增加拼写题和听音题。
- 答错词进入错词强化队列。
- 拼写错和听音错需要分别标记，便于后续专项训练。

## 复习策略

第一版使用简化的熟练度和间隔复习策略，不追求复杂算法。

- 新词初始熟练度为 0。
- 答对提升熟练度，并延后下次复习时间。
- 答错降低熟练度，并缩短复习间隔。
- 连续答错的词进入高优先级错词队列。
- 拼写题错误标记为拼写薄弱。
- 听音题错误标记为听辨薄弱。

后续可根据真实使用数据逐步改进为更精细的遗忘曲线模型。

## 家长端设计

家长端第一版保持轻量：

- 切换两个孩子档案。
- 查看今日任务是否完成。
- 查看正确率、连续学习天数和闯关进度。
- 查看高频错词和薄弱类型。
- 给孩子选择内置词表。
- 导入自定义词表。
- 导出和导入本地备份。

## 词库设计

### 内置词表

第一版预留内置词表能力，可先放入少量示例词表，后续逐步扩充：

- 小学高年级基础词表。
- 初中核心词表。
- 高中核心词表。

### 自定义导入

家长可以通过粘贴文本或文件导入词表。推荐字段：

- 单词。
- 中文释义。
- 音标。
- 词性。
- 例句。
- 标签。
- 来源。

第一版可以先支持 CSV，后续增强 Excel 导入。

## 数据策略

### 本地优先

- 不依赖网络即可学习。
- 每个孩子的数据保存在本地。
- 备份导出为结构化 JSON 文件。
- 备份导入需要校验版本、数据完整性和孩子档案冲突。

### 预留云同步

数据访问必须通过 Repository 接口，不让界面直接依赖具体存储实现。

后续云同步可新增远端数据源，并在 Repository 中处理：

- 本地数据读取。
- 远端数据拉取。
- 冲突合并。
- 同步状态。
- 错误恢复。

## 技术路线

### 推荐技术栈

- 框架：Flutter。
- 语言：Dart。
- 第一阶段：Flutter Web / PWA。
- 后续平台：Android、iOS、macOS、Windows。
- 状态管理：Riverpod。
- 数据层：Repository 抽象。
- 本地存储：先从轻量本地存储开始，后续切换或扩展到 Drift / SQLite。

### 架构原则

- 学习核心逻辑使用纯 Dart 实现，便于测试和跨端复用。
- 平台能力单独封装，例如文件选择、备份导入导出、音频播放和通知。
- 界面层只负责展示和用户交互，不直接修改持久化数据。
- 数据模型需要包含版本字段，方便后续迁移。

## 核心实体

- ChildProfile：孩子档案。
- WordEntry：单词。
- WordBook：词表。
- StudyTask：每日任务。
- ReviewSchedule：复习计划。
- AnswerRecord：答题记录。
- MistakeRecord：错词记录。
- LevelProgress：闯关进度。
- BackupPackage：备份数据。

## 成功标准

- 两个孩子可以独立完成每日任务。
- 家长能在 30 秒内知道每个孩子今天学得怎么样。
- 自定义词表可以导入并用于学习。
- 错词能被自动纳入后续复习。
- 数据可以导出备份并重新导入。
- 第一版代码结构支持后续接入云同步，不需要推翻重写。
